Peer-review

By Wieger Wittrock
Oct 25th, 2021

I reviewed the website of Hugo van't Riet. You can visit his website here.

Review

Hugo's website was really good, it had a lot of information about himself and about his projects. The site immediately starts with personal information and a description, which I like. The design is consistent through the whole website, and it has personal design elements. One of the only things I did not like about the design was the grey background everywhere. The background makes it a bit difficult to know where you are on the website (maybe the pages could have different backgrounds). But the rest of the navigation works really well since the blog and the portfolio have there own seperate page.
There are already two blogposts on Hugo's portfolio page, they have a lot of information. He also uses many images inside the posts, which I like. The only thing that was still missing was a way to contact Hugo, but that could also be because I am reviewing his website on monday already when we need to hand it in on friday :)

Reflection

From reviewing I learnt some new cool things about design. I really liked the way Hugo had everything centered in his website, it makes it easy to read everyting. Also after looking in detail at the way he made his navigation bar, I learnt that when you put everything in the middle, you can scale the screen without really big overlapping of texts.

Analysis

The analysis is about the home/index page of Hugo's website. I especially like the looks of the navigation bar. By pressing Ctrl+U, we can see that Hugo's used "plain" HTML and CSS to make his website (no WordPress), which is good! He also didn't use a template.
By looking at the code and using the inspect function we find that the navigation bar is an unordered list. Every link is inside a list-element. In CSS we can see how he styled the navigation bar:
First, he uses "list-style-type: none", I actually didn't know what this exactly did. I looked it up and it removes the circles, that normally show that something is a list. Very clever! All the links inside the navigation bar are displayed as a block, so they are clearly seperated. He also uses "overflow: hidden;" so text will not be visible outside the boundaries.
The rest of the CSS is mostly used for design like colors and font. The rest of the page is text and a picture. He didn't use a gallery or JavaScript/JQuery/plugins.

Personal touch

By Wieger Wittrock
Oct 18th, 2021

This blog post is about the personal touch of this website. I started from scratch and did not use a template. So everything I made, was made with a personal touch on this website.

The first code I want to show you that contributed to the personal touch of this website, is showcased in this image. My header consist of 3 different buttons and a big home button. When you scale the screen-size down, then all the buttons will keep floating on their side of the screen: "portfolio" stays left, and "home" and "blog" stay to the right. The big home button will stay in the center.
If you scale the screen-size too far down however, then the buttons will overlap each other. The code from the picture prevents this. If the screen-size is smaller then 600px, then some stuff will happen.
First off, the buttons will no longer float to the right, or the left.
The buttons are now displayed as blocks, so instead of standing next to eachother the buttons now will be under eachother.
Also, the text in the buttons is now alligned to the left instead of the middle, because I thought that looked nicer.
Finally, all the images in the blog posts will stop floating and will also be displayed in blocks, simply because there is not many space left if all the images will be next to the text.

The second piece of code is about the background color of the website. As you can see behind this blog post, the background color is not a single color. Instead it is a linear-gradient from four colors. I chose these four colors, (Electric Blue, Celeste, Spring Green and Emerald) because I like them and because I think they fit with eachother as a background. The gradient goes from the top-left corner to the bottom-right corner. I also made a background color for browsers that do not support the gradient.
The usefull thing of css is that now the background color will be the same for every page on this website, by only using one piece of code!

Update on video project

By Wieger Wittrock
Oct 13th, 2021

This is an update on how it is going with our video project! For We Create Identity we have to make a video where the viewer can choose which path the video will go on. This is a really cool idea, and our group is making a lot of progress already. A very important part of the video is of course the story and the choices the viewer can make. Our idea was to film a reunion of highschool friends, they will have a fancy dinner at someones place. One of the friends however doesn't have a fancy suit and the rich host (me!) is a bit arrogant to the poor friend.
What the host doesn't know is that the poor friend has a sick child. In desperate need of money, the poor friend steals from the rich host, but the main character (who knows about the situation of the poor friend) notices the theft...
From this point on, the viewer can chose to help to poor friend stealing, tell the host, try to resolve the problem yourself or even call the cops.

Yesterday we had a meeting with a teacher about our story, we changed a bit of the choices but the main story was alright! We also made a story tree which you can see here.
We are now still working with story improvements, and we started a bit with writing the script.

Website Review

By Wieger Wittrock
Oct 10th, 2021

In this blog post I will review the website: bol.com.
Bol.com is a webshop in the Netherlands and offers general merchandising products in many categories. The website has a lot of users, mainly by the good service, many different products to buy and the short delivery time. As of 2020, the store serves 11 million active customers in the Netherlands and Belgium and offers over 23 million items. Since 2011, bol.com has also opened its webshop for retailers to sell and effectively became a platform. Since the introduction of "Selling via bol.com", more than 30,000 retailers have already sold via bol.com. The company employs approximately 2,000 workers.

    Strong points
  • Nice header, where you can find all catagories
  • You login automatically
  • The homepage has a clear layout
  • There is a clear account overview
  • The website remembers your products
    Weak points
  • The background is boring
  • You lose many button when scaling the screen down
  • They make a lot of advertisements as links
  • Complicated subscription page
  • You can only compare a maximum of 5 products
